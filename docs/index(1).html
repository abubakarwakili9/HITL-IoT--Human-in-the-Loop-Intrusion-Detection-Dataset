Abstract

Human oversight has become a central requirement for deploying trustworthy AI systems, reinforced by regulatory frameworks such as the EU Artificial Intelligence Act. Yet existing IoT intrusion detection datasets provide only benign and attack labels, offering no insight into how human analysts review alerts, express confidence, or justify their decisions. This absence of human decision data limits progress in developing transparent, accountable, and human-centred intrusion detection systems.

We introduce HITL-IoT, the first benchmark dataset explicitly designed to support research on human-AI collaboration and oversight in IoT cybersecurity. The dataset includes structured annotations from security professionals at expert, intermediate, and novice levels. Each annotation records the analyst's decision, confidence score, decision latency, and justification text, creating competence-aware ground truth unavailable in prior IoT IDS benchmarks.

HITL-IoT spans twelve IoT attack categories across the cyber kill chain, enabling comprehensive evaluation of human-AI decision workflows under realistic adversarial conditions. To demonstrate dataset utility, we provide baseline results across classical machine learning and deep learning models using standardized preprocessing and evaluation protocols.

HITL-IoT supports research on:

  • Human-centred intrusion detection
  • Expertise-aware alert routing and selective deferral
  • Trust and confidence calibration
  • Empirical analysis of oversight mechanisms required by EU AI Act Article 14

All dataset files, metadata, and reproducible tools are publicly released to support transparent and extensible research. HITL-IoT establishes a foundational resource for advancing trustworthy, human-aligned cybersecurity solutions for IoT systems and digital infrastructure.

Dataset Architecture

Generation Pipeline

HITL-IoT is generated through a four-module pipeline that creates realistic IoT traffic, extracts ML-ready features, simulates human analyst behavior across expertise levels, and exports versioned, cryptographically-verifiable datasets.

Figure 1: Four-module architecture for HITL-IoT dataset generation. Module A simulates realistic IoT traffic with 12 attack families. Module B extracts 54 CICFlowMeter-style features and generates ML predictions with calibrated confidence. Module C implements expertise-aware human annotation with confidence scores and decision latency. Module D exports versioned datasets with integrity verification.

Module A

IoT Environment & Traffic Modeling

  • Generates realistic benign + adversarial IoT traffic
  • 12 attack families across cyber kill chain
  • Smart home topology (cameras, thermostats, speakers, etc.)
  • Ambiguity controls for realistic uncertainty

Module B

Flow Representation & ML Confidence

  • 54 CICFlowMeter-style features per flow
  • Device-aware deviation scoring
  • ML probability with calibrated uncertainty
  • Ambiguity score for selective sampling

Module C

Human-in-the-Loop Oversight

  • Ambiguity-aware sampling (selects top 8% flows)
  • Human annotators at 3 expertise levels
  • Records: decision, confidence, latency, justification
  • Competence-aware decision model

Module D

Dataset Export & Reproducibility

  • Export to CSV, Parquet, annotation CSV
  • Metadata JSON with schema and taxonomy
  • SHA-256 integrity checksums
  • Version control (v1.0)

Network Topology

The dataset captures traffic from a realistic smart home environment with 12 IoT device types, including IP cameras, smart thermostats, doorbells, speakers, and smart TVs. Traffic flows through a home Wi-Fi router with an IDS sensor at the mirror port, capturing both benign connections to legitimate cloud services and malicious traffic from attacker infrastructure.

Figure 2: Smart home network topology showing legitimate cloud services (blue arrows) and attacker infrastructure (red dashed arrows). The IDS sensor captures bidirectional flows at the gateway, enabling comprehensive attack detection across reconnaissance, lateral movement, data exfiltration, and device hijacking scenarios.
